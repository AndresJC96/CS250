# CS 250 Portfolio Reflection

## How do I interpret user needs and implement them into a program? How does creating user stories help with this?

I interpret user needs by understanding what the user is trying to accomplish before thinking about the technical solution. Creating user stories helps break those needs into smaller, manageable tasks that focus on the user's perspective. This makes it easier to prioritize features and ensures the software provides value to the end user.

## How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I approach development by first understanding the requirements, creating a plan, and then building the program one step at a time. Throughout this course, I learned the value of Agile practices such as sprint planning, daily communication, sprint reviews, and retrospectives. In future projects, I plan to continue using these practices because they encourage continuous improvement and make it easier to adapt when requirements change.

## What does it mean to be a good team member in software development?

Being a good team member means communicating openly, completing assigned work on time, and being willing to help others when needed. It also means accepting feedback and using it to improve. Agile development depends on collaboration, so contributing to the success of the entire team is just as important as completing individual tasks.
